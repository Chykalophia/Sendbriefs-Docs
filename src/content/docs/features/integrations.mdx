## How integrations work today

There are two separate integration patterns under **Settings → Integrations**:

### A) Integration credentials

The **Integrations** card lets you connect three providers:

- **Google** — OAuth ("sign in with Google" flow). Asks for read access to Google Analytics and your profile info.
- **Slack** — OAuth flow. Asks for permission to post messages, list channels, and read user info.
- **ClickUp** — personal API token. Paste a token starting with `pk_` from ClickUp Settings → Apps.

All three credentials are encrypted at rest (industry-standard AES-256) and scoped to the workspace. You can revoke any of them at any time.

### B) Brief integrations (the pipeline that actually runs)

The **Brief integrations** card lets you create *mappings* — pairs of (brief template + connected integration + source selector). When a new item appears in the source, it's auto-appended as a block to the matching draft brief.

This is the ambient-ingest pipeline. Only one provider has a working data adapter today (ClickUp).

## Actually working today

### ClickUp ambient ingest

**The full path:**

1. **Settings → Integrations → Integrations** card → **Connect** on the ClickUp tile.
2. Paste a personal API token (tokens start with `pk_`; generate from ClickUp's Settings → Apps page). We check the token works before saving.
3. **Settings → Integrations → Brief integrations** card → **Add mapping**. Pick a brief template, pick the ClickUp integration, choose the list/source.

After save, the ambient ingest job pulls items from the ClickUp source on a schedule (production: Upstash QStash) and appends them as blocks to the matching brief template's current-period draft. Items already ingested are tracked so re-runs don't duplicate.

**Status indicators (on the mapping row):**

- **Active** — mapping is enabled; the job will pull on its next tick.
- **Paused** — mapping exists but the job skips it.
- **Error** — last run failed; check the detail on the row.

**Gotcha — schedule registration is a deploy step.** Even with a mapping created, the background job has to be registered on our side. If your workspace has mappings but nothing is ingesting, the schedule may not be wired up — let support know.

## Connection-only (no functional data flow yet)

These are visible in the **Integrations** card. The OAuth flows complete; credentials are stored. But the product doesn't currently use them for anything user-facing.

### Google

**What the marketing card suggests:** Google Analytics data flowing into briefs.

**What actually happens after you connect:** Credentials are stored against your workspace. No code path currently uses them — there is no GA4 data fetch in the product as of this preview. The OAuth flow exists so that the eventual Google-data-into-briefs adapter will already have credentials when it ships.

If you connect Google, expect a green **Connected** badge and the email address of the connected account. Don't expect GA4 data to appear in briefs as a result.

### Slack

**What the marketing card suggests:** Sending brief summaries to Slack channels.

**What actually happens after you connect:** Same as Google — credentials are stored, no other code path uses them. There is no Slack-send code in the product today. The Brief integrations card lists Slack as a connected source, but the Slack adapter for ambient ingest isn't built either.

## Where this lives in the app

All integration management is under **Settings → Integrations**. Two cards live there:

- **Integrations** — the credentials card. Available providers: **Google** (OAuth), **Slack** (OAuth), **ClickUp** (personal API token). Connect, see status, disconnect.
- **Brief integrations** — per-template ambient-ingest mappings. Pick a brief template, pick a connected integration, enter a source selector. The mapping starts polling and appending new items into the matching draft brief.

Both cards are visible to Owners and Admins (`settings:update` permission).

## Security and credentials

- **All credentials** (OAuth tokens and API tokens alike) are encrypted at rest with industry-standard AES-256, scoped to the workspace.
- **Every connection is logged** — we record who granted access, when, and which service.
- **Disconnecting** any integration revokes the stored credential immediately.
- **Deleting a Brief integrations mapping** removes the mapping only — the underlying credential stays connected.

## Common gotchas

- **The two cards are separate.** Connecting Google in the credentials card doesn't do anything useful by itself — you'd still need a Brief integrations mapping AND a working data adapter for that provider.
- **Only ClickUp actually ingests today.** Google and Slack are connect-only — creating a Brief integrations mapping that points at them does nothing until their adapters land.
- **OAuth tokens expire.** Google's refresh-token flow is wired. Slack tokens currently have no refresh path — if revoked, reconnect.
- **There's no per-client integration scope.** Integrations are workspace-level. Brief integrations mappings wire to a *template*, not a client — a template can be used across multiple clients, but each mapping has one source selector.
