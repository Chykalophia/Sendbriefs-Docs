<strong>/activity</strong> in the sidebar is a three-tab page: <strong>Workspace Activity</strong> (recent business events in plain language), <strong>Email Delivery</strong> (recent sends and their per-recipient status), and <strong>Advanced</strong> (the full audit log with filters by action, resource, user, and date). Settings → Logging is a link card that points here.
</aside>

## Why this page exists

Every meaningful action in the workspace — a brief sent, a member invited, an integration connected, a role changed, a setting toggled — writes a row to `audit_logs`. The Activity page is how you read those logs.

It's also where Email Delivery rolls up: per-send data from SendGrid's event webhook, surfaced in one searchable place rather than per-brief.

## The three tabs

### Workspace Activity

A reverse-chronological feed of "things that happened" in plain language:

- "Sam invited alex@example.com as Admin."
- "Acme Q1 Brief was sent to 4 recipients."
- "Workspace name changed from 'My Studio' to 'My Studio Inc.'"

Click any entry to drill into the detail. Useful for a quick "what changed this week" scan.

### Email Delivery

A roll-up of send events. Shows recent sends with:

- Brief title.
- Client.
- Recipient count.
- Aggregate delivered / opened.
- Sent timestamp.
- A link into the brief's delivery card for per-recipient detail.

This is the view to use when a client says "I never got the brief" — you can confirm send + delivery from one place rather than opening every brief individually.

### Advanced (raw audit log)

The unprocessed log. Filters by:

- **Action type** — things like "report sent", "report approved", "client created", "workspace settings updated", "integration connected", "scheduled report ran", and 40+ more. The filter dropdown shows the raw action names (e.g., `report.send`) so you can match exactly.
- **Resource type** — workspace / user / client / contact / report / template / integration / etc.
- **User** — who performed the action.
- **Date range** — filter to a window.
- **Resource ID** — find every action on a specific brief or client.

Each row shows:

- Timestamp.
- Action.
- Resource type + resource ID.
- Acting user.
- IP address and user agent (helpful for security audits).
- Metadata (action-specific JSON payload).

Click any row to see the full metadata blob.

## Who can see what

- **Workspace Activity** and **Email Delivery** are available to all workspace members.
- **Advanced** (raw audit log) requires admin-level access (Owner or Admin role, `audit:read` permission in practice).

## What's logged

50+ action types in all. The main categories:

| Category | Examples |
|---|---|
| **Auth** | Login, logout, password changed, MFA enabled / disabled |
| **Workspace** | Created, settings updated, deleted |
| **Members** | Invited, accepted invitation, role changed, removed |
| **Clients** | Created, updated, archived, deleted |
| **Contacts** | Created, updated, deleted, suppressed, restored |
| **Briefs / Reports** | Created, edited, submitted, approved, changes requested, sent, resent, archived, deleted |
| **Templates** | Created, updated, cloned, deleted |
| **Schedules** | Created, ran (success / fail), enabled / disabled, deleted |
| **Integrations** | Connection granted, revoked; token added, removed |
| **Email settings** | Sender changed, signature changed, locked / unlocked |
| **Billing** | Subscription created / updated, status changed, payment failed, add-on purchased |
| **GDPR** | Data export requested, data erasure requested |

## Retention

Audit log retention is configurable per workspace. The default is 90 days, but Enterprise and higher tiers can extend. After that, old entries are deleted.

For longer compliance retention, the audit log can be exported (ask support — there's no self-serve "export everything as CSV" button yet).

## Common queries

**"Did this brief send to everyone?"** — Email Delivery tab → find the brief → check per-recipient status.

**"Who removed this client?"** — Advanced tab → filter resource type = client, resource ID = the client ID. Look for `client.delete` action.

**"Who changed billing?"** — Advanced tab → filter resource type = workspace, action type = `billing.*`.

**"Has any new login happened from outside our IP range?"** — Advanced tab → filter action type = `auth.login`. Sort by IP. Spot anomalies.

**"What did this admin do last quarter?"** — Advanced tab → filter user = the admin, date range = last quarter.

## Settings → Logging

The Logging tab in Settings is a thin pointer to this page. It's not an automatic redirect — open it and you'll see a card with a button labeled "View Activity & Logs →" that opens `/activity?tab=advanced`.

Why have both? The Settings tab keeps the in-Settings navigation consistent (every category has tabs); the `/activity` page is reachable from anywhere via the sidebar. Same destination.

## Common gotchas

- **Bulk operations log as one row, not N rows.** A CSV contact import that adds 50 contacts shows up as one audit log entry with the count in metadata, not 50 entries.
- **The Advanced tab paginates.** Default page size is 10 rows. Filter before scrolling — paging through 30,000 rows is unhelpful.
- **Metadata size cap.** Per-row metadata is capped at 100 KB. Actions with larger payloads (e.g., a huge import) store a summary, not the full payload.
- **Soft-deleted resources may not show their post-delete actions.** If you archive a client and then archive briefs under that client, the brief actions log; the resource itself stays in the audit log even after archive.
